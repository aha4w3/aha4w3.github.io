Frequently Asked Questions

  • What are the advantages of Node.js?

    Node.js is asynchronous and event-driven. All APIs of Node.js library are non-blocking, and its server doesn't wait for an API to return data. It moves to the next API after calling it, and a notification mechanism of Events of Node.js responds to the server from the previous API call. Node.js is very fast because it builds on Google Chrome's V8 JavaScript engine. Its library is very fast in code execution. Node.js is single threaded but highly scalable. Node.js provides a facility of no buffering. Its application never buffers any data. It outputs the data in chunks.

  • How about the safest way to use MetaMask?

  • Which networks does MetaMask support?

    The default network for all MetaMask wallets is the Ethereum mainnet. However, it is also able to connect MetaMask to other blockchain networks, such as: Polygon, Fantom, Binance Smart Chain, Avalanche, Arbitrum, Cronos, Harmony ONE, Velas, HECO, and xDAI. You can find out more about EVM-compatible networks by google search.

  • Why is PostgreSQL one of the advanced RDBMS?

    Following are the major highlights which make PostgreSQL one of the advanced RDBMS - (1)Supports both basic indexing (B-tree, Multicolumn, Expressions, Partial) and advanced indexing (GiST, SP-Gist, KNN Gist, GIN, BRIN, Covering indexes, Bloom filters) which makes the read/write operations much faster than the traditional DBMS. (2)Since it's an object-based relational DBMS so it supports table inheritance, extending data types etc. (3)Maintains data consistency using a multi-version model — MVCC (Multi-Version concurrency Control). This means that while querying a database each transaction sees a snapshot of data (a database version) as it was some time ago, regardless of the current state of the underlying data. This protects the transaction from viewing inconsistent data that could be caused by (other) concurrent transaction updates on the same data rows, providing transaction isolation for each database session. Thus, it ensures data don’t conflict and the read operation doesn't block the write operation and vice-versa.[Ref -https://www.postgresql.org/docs/] (4)Support for major programming languages (such as C/C++, JavaScript, Java, Python, R, Go, .Net, etc) gives added power to the developers. (5)Allows Point-in-time recovery (PITR) to restore databases to a specific moment in time and Write ahead log (WAL) that logs all changes to the database using tools such as pgBackRest. (6)Has an open source license which means it carries no licensing cost and is free to use.
